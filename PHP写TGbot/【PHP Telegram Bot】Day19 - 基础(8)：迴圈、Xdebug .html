【PHP Telegram Bot】Day19 - 基础(8)：迴圈、Xdebug

当需要重复做类似的事情时，不需要写很多行重复的程式，用迴圈就能一次执行很多次

迴圈

while

如果符合小括號中的条件，就执行大括號中的程式，並且执行完后再次检查条件，直到条件不符合时结束

通常在不知道会执行几次时使用

例如我想把 1000 以內，所有 2 的次方数印出来

$i = 1;
while (2 ** $i < 1000) {
    echo 2 ** $i, "\n";
    $i++;
}

do while

一样是不知道会执行几次时使用，但这次是先做大括號中的程式，才检查小括號的条件

有时候不管有没有符合条件，都要执行一次程式，这种方式就很好用

$i = 1;
do {
    echo 2 ** $i, "\n";
    $i++;
} while (2 ** $i < 1000);

for

通常是知道执行几次时使用，for 会稍微复杂一点，有四个区域，A、B、C 都能留空

  1. A：把 0 存进变数 $i
  2. B：逻辑运算，结果如果是 true 就执行大括號中的程式，否则结束迴圈
  3. D：执行大括號中的程式
  4. C：D 结束后会先执行 C
  5. B：之后又回到 2.，就这样周而復始

其实 while 和 for 是通用的，不管是哪种形式两者都能修改成另一个，这个就交给你自己试试啦

function a($i, $x) {
    echo $x;
    return $i;
}
for (a($i = 0, 'A'); a($i < 3, 'B'); a($i++, 'C')) {
    a($i, 'D');
}
// ABDCBDCBDCB

foreach

这个比较特別一点，通常会跟阵列做搭配，把阵列中的东西一个个取出来处理

它有两种写法，就看要不要使用到阵列的 key

$a = ['a' => 'air', 'b' => 'bag', 'c' => 'cat'];

foreach ($a as $key => $value) {
    echo "key: $key, value: $value\n";
}

foreach ($a as $value) {
    echo "value: $value\n";
}

另外，阵列的值可以使用 & 参考 Call By Reference（键不行），改写原本的阵列

$a = ['a' => 'air', 'b' => 'bag', 'c' => 'cat'];
foreach ($a as &$value) {
    $value = strrev($value);
}
print_r($a);
// [a] => ria
// [b] => gab
// [c] => tac

continue、break

上面四种迴圈都拥有这两个关键字

continue 可以直接跳过接下来的程式

function a($i, $x) {
    echo $x;
    return $i;
}
for (a($i = 0, 'A'); a($i < 3, 'B'); a($i++, 'C')) {
    continue;
    a($i, 'D'); // 不会执行
}
// ABCBCBCB

break 可以结束迴圈，但是只有一层

function a($i, $x) {
    echo $x;
    return $i;
}
for (a($i = 0, 'A'); a($i < 3, 'B'); a($i++, 'C')) {
    break;
    a($i, 'D'); // 不会执行
}
// AB

双重迴圈

如果想在二维阵列中找东西，就会用到双重迴圈，但是这样写是没办法跳出两层迴圈的

$a = [
    ['0', '1', '2'],
    ['a', 'b', 'c'],
    ['air', 'bag', 'cat']
];
foreach ($a as $b) {
    foreach ($b as $value) {
        echo "\n", $value;
        if ($value === 'b') {
            echo ' 找到 b 了';
            break;
        }
    }
}
// 0
// 1
// 2
// a
// b 找到 b 了
// air
// bag
// cat

有个关键字 goto 但不建议使用，我有另一个方法：借用函式和 return

function findValueInMatrix($something, $array) {
    foreach ($array as $b) {
        foreach ($b as $value) {
            echo "\n", $value;
            if ($value === $something) {
                echo " 找到 $something 了";
                return true;
            }
        }
    }
    return false;
}
$a = [
    ['0', '1', '2'],
    ['a', 'b', 'c'],
    ['air', 'bag', 'cat']
];
findValueInMatrix('b', $a);
// 0
// 1
// 2
// a
// b 找到 b 了

Xdebug

Xdebug 是 PHP 的除错工具，它是一个需要自己下载的扩充元件，可以让你看清楚程式的执行过程，解决一看到迴圈脑袋就当机的问题

首先要安装 VS Code 的扩充功能，因为我们要在 VS Code 中执行 Xdebug

  • PHP Debug

然后到 Xdebug 的官网，找到安装的方法

  • https://xdebug.org/docs/install

Ubuntu or Linux

sudo apt install php7.4-xdebug

MacOS

pecl install xdebug

如果是 M1 版本，你可能需要使用以下指令

arch -x86_64 sudo pecl install xdebug

Windows

在官网找出 Windows 载点：https://xdebug.org/download


选择对应的 PHP 版本，TS 表示是线程安全（Thread Safe）

  • php-7.4 x64 Thread Safe (xdebug-3.1.0beta2)
  • php-7.4 x86 Thread Safe (xdebug-3.1.0beta2)

闭著眼睛按保留

下载后放到 ext 资料夹里

之后要设定 php.ini，在最底下贴上这段

[Xdebug]
zend_extension = "path-to-Xdebug.dll"

填上路径（填你自己的）

最后按储存就完成啦

使用 Xdebug

先开启你的程式档案，然后在你要停止的行號左边点一下，就会出现一个小红点

在 VS Code 左边有个像这样的 Icon，找到它並点下去

按下「建立 launch.json 档案」，这样才不用每次都要重选

之后就会自动建立一个侦错的设定档並且开启，不用理它直接关掉就对了

点开选单，选择「Launch currently open script」，这个是 CLI 模式时的侦错，另外两个是给伺服器用的

按下这个按钮（或是按 F5）就会开始侦错你开启的那个档案

停在这里表示还没开始执行这一行

左边可以看到 $i 还没被赋值

按下「逐步执行」（或是按 F11）

你可以发现 $i 的值变成 0 了，但程式还是在同一行

因为这一行会执行两个步骤，赋值 $i = 0 和检查 $i < 10

再次按下「逐步执行」（或是按 F11），才会执行 $i < 10

Xdebug 功能

先看这排按钮，最左边是让你能拖动它的

从左到右依序是

按钮 说明
继续执行程式直到下一个断点，或是程式结束
执行一步程式，遇到函式就只当做一步，不会跑进函式
执行一步程式，会跑进函式
直接执行到函式结束，会停在函式后的下一步位置
重新执行侦错，从头开始执行程式
中止侦错，直接停止程式，不再执行

变数 VARIABLES

显示当前作用域的变数

堆叠 CALL STACK

可以让你看到现在程式的位置，並且可以查看呼叫这个函式的地方

监看 WATCH

在这边可以对变数做简单的运算，例如我想知道 $i 的二次方是多少

断点 BREAKPOINTS

打勾表示你要停下来的地方，在这边可以看到哪个档案有断点，还有断点在第几行

甚至可以编辑停止的条件
