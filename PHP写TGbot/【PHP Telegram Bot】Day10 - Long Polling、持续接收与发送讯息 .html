【PHP Telegram Bot】Day10 - Long Polling、持续接收与发送讯息

用 PHP 打造专属於自己的 Telegram 聊天机器人吧！ 系列 第 10 篇

25 天前 ‧ 78 瀏览
0

昨天有讲到一个叫做 getUpdates 的方法，但我们没有填任何参数，今天要利用 offset 与 timeout 达成 Long Polling

Long Polling

先再执行一次昨天最后的程式，你会看到有个叫做 update_id 东西

把它填到 offset 的值，然后 timeout 先填 60，表示 60 秒以內 TG 伺服器要回应你

填好之后按下执行，你会发现和刚刚拿到的东西一模一样，这是因为它会从你设定的 update_id 开始取讯息

我们把刚刚填的值加一，再执行看看

你会发现程式跑了整整一分钟才跑完，而且啥都没有，只有印出一个空阵列 Array()

这是因为 540834396 就是最后一则讯息了，並没有 540834397 这则讯息

再执行一次程式，这次执行之后私讯机器人一则讯息

你会发现发送讯息后，TG 伺服器立刻就回应了，而且程式也马上跑完了

这个就是 Long Polling

发出请求时，伺服器会一直到有新讯息时才会回应，或是到达时间限制（timeout）时回应空讯息

这样不仅不用一直发送请求，而且有新讯息时也能立刻收到

我们再来试个东西，如果把 540834397 改回 540834396 看看会发生什么事

按下执行后，你会发现 540834396 不见了，只剩 540834397

这是因为 TG 伺服器会把小於 offset 的 update_id 都当作是你已经处理过的了，並且將这些讯息捨弃，所以你就没办法再次读取了

处理收到的讯息

我们要用到 processMessage 这个官方范例里的一个函式，这个函式里要填的是 message，但是要怎么把它取出来呢？

首先第一步，把 print_r() 刪除，改成 $response =，记得后面的括號要刪掉一个

把它照著层数接在 $response 的后面，message 是字串，所以要用引號 ' 包起来

接好之后会像这样

最后把 processMessage() 加上去，別忘了后面的小括號 ) 和分號 ;

按下执行后你会发现它印出了一坨东西

{"chat_id":127355800,"reply_to_message_id":24,"text":"Cool","method":"sendMessage"}

这是因为它用了第三种发送讯息的方式，但现在程式並不是在 Webhook 模式下运作，所以就直接 echo 到终端机上了

我们只需要將第 139 行的 apiRequestWebhook 改成 apiRequest 就行了

改完再次执行程式，此时 TG 「叮咚」了一声

成功用官方的函式处理了讯息

处理每则讯息

其实不只是 $response[0] 里面有 message，$response[1], $response[2]... 也会有 message，至於会到数字多少，这取决於一次取到了多少则讯息

如果你发送了两则讯息才执行一次程式，此时就会出现两个 message，那要如何每个 message 都处理到呢，这时候就要用到 foreach 这个函式，它会一个个的把里面的东西交给你，直到全部完成为止

来稍微修改一下刚刚的程式，把 $response 交给 foreach，中间的 $update 就是 $response[0], $response[1]...（你可以用 print_r($update); 印出来看看），然后把 $update['message'] 填进 processMessage() 函式中，记得加分號~

这样每则讯息就都会处理到了

让程式持续运作

聪明的你应该已经发现了，不管执行几次它都会重复处理讯息，因为它总是从 offset 开始处理

每次执行一次程式就要改一次 offset，这样多麻烦，当然是要让程式自己设定

首先要找出这很多则讯息的最后一则讯息，我们可以用 count($response) 知道总共有几则讯息，count($response) - 1 表示最后一则（因为是从 0 开始数），我们先將这个值存进一个叫做 $latest 的变数里

然后把 $latest 填进 $response 的第一个框框里，第二个框框填上 'update_id'，update_id 是字串，所以要用引號 ' 包起来

这样就取得了最后一则讯息的 update_id

我们是要把 offset 的值填下一则讯息的 update_id，这样才不会一直重复读取同一则讯息，这边只要在后方多打个 + 1 就行了，之后把计算的结果存进 $update_id 变数里

还记得会有没取到讯息的时候对吧，没取到讯息时 $latest 的值会变成 -1，但是框框里不能出现负数，这时就要加个 if 条件 $latest !== -1，有读取到讯息时才把 update_id 取出来

再来就是把 $update_id 填到 offset 的值里，还有加上 while (true) {}，让程式不断重复执行

最后的最后，在最前面加上一行 $update_id = 0;

这样就大功告成啦

执行程式后，试著跟机器人说 Hi、Hello 吧

对了，如果想要停止程式的话，就对著终端机按下 Ctrl + C，程式就会停止了

结尾

今天的內容可能有点困难，看不太懂没关係

如果没办法照著做出来的话，欢迎直接复製下方程式码来玩玩：

$update_id = 0;
while (true) {
    $response = apiRequest("getUpdates", array('offset' => $update_id, 'timeout' => 60));
    foreach ($response as $update) {
        processMessage($update['message']);
    }
    $latest = count($response) - 1;
    if ($latest !== -1) {
        $update_id = $response[$latest]['update_id'] + 1;
    }
}
